Software Documentation

Benefits

We design, write, update and maintain software application documentation for organisations in Victoria. Our writers have a background in IT, so we understand the issues faced with software development, implementation and support.

Implementation of new software applications

If you are implementing a new software application our services can provide documentation for your:

  • Implementation Staff
  • End Users
  • Operational Users
  • Instructors
  • Implementation staff may need assistance converting 'IT speak' into 'user speak'. Knowledgedoc can assist your staff in converting technical documents such as flow charts, design and functional specifications into a format your users can understand.
  • Knowledgedoc can provide accurate, consistent and easy to use documentation so your end users can operate your software efficiently from day one.
  • Operational end users may need policies and procedures or standard operating procedures (SOPs) so they can 'get it right, all the time'.
  • Instructors need training materials that are written for the audience, so that the training is effective.

Implementing existing software applications

If you are implementing an existing software application your:

  • End Users
  • Support Staff
  • Instructors
  • Managers
  • New end users need accurate, consistent and easy to use documentation so that they can operate efficiently from day one.
  • Support staff need technical and support information so that they can support the end users well.
  • Instructors need training materials that are written for the audience, so that the training is effective.
  • Managers need to capture and protect the knowledge held in the heads of critical support staff.

Software Development Companies

If you are a software development company, you need documentation to:

  • make your product look professional
  • minimise support costs
  • free you and your development staff to do what you do best.

Documentation for software applications

We can assist you with all aspects of documenting software applications, including:

  • user manuals
  • online help
  • context-sensitive help
  • web-based help
  • functional design and detailed design documentation for developers
  • network architecture documentation
  • training materials
  • technical support documentation
  • web content
  • policies and procedures.
